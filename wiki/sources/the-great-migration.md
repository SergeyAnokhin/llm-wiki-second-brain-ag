# The Great Migration

**Источник:** The Great Migration  Home Assistant Developer Docs.md
**Дата добавления:** 2026-05-05
**Тип:** статья

## Краткое содержание

Исторический документ (от февраля 2019 года), написанный основателем системы [[Paulus Schoutsen]]. В нём объясняется архитектурный сдвиг в версии HA 0.87, названный "The Great Migration".

Суть миграции заключалась в переходе от платформенно-ориентированной архитектуры файлов (например, `light/hue.py`) к компонентно-ориентированной (например, `hue/light.py`). Это позволило сгруппировать все файлы одной интеграции в одну папку, что значительно облегчило поддержку масштабных проектов.

## Ключевые утверждения

- Размещение файлов как `light/zwave.py`, `switch/zwave.py` приводило к "maintenance hell" (аду поддержки), поэтому архитектуру изменили на группировку по домену: `zwave/light.py`, `zwave/switch.py`.
- Частичное переопределение ([[Custom Component Overlays]]) встроенных компонентов запрещено. Если разработчик хочет изменить встроенный компонент через папку `custom_components`, он обязан скопировать в неё **все** файлы этой интеграции, а не только один `sensor.py`.
