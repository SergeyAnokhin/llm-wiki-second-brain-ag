# Camera Archiver Repository

**Источник:** https://github.com/SergeyAnokhin/camera_archiver
**Тип:** Исходный код (пользовательский компонент)

## Краткое содержание
Это репозиторий вашего старого пользовательского компонента `camera_archiver` для Home Assistant. Проект представляет собой комплексное решение для архивации медиафайлов с камер (загрузка по FTP, MQTT, из локальных папок) и их последующей обработки (отправка в Elasticsearch, вызов API). 

## Для чего нужен этот проект?
- Он решает узкоспециализированную задачу управления архивами записей с камер видеонаблюдения внутри умного дома.
- В рамках нашей базы знаний этот репозиторий служит **отправной точкой (legacy code)**. Он демонстрирует ваш прошлый стиль программирования (синхронные методы, сложная валидация `configuration.yaml` через `voluptuous`, использование старого механизма загрузки платформ).
- Его главная ценность сейчас — это возможность на его примере отработать рефакторинг: перевод компонента на современную асинхронную архитектуру с использованием `Config Flow` и `Options Flow`.

## Интересные находки
- Глубокое использование библиотеки `voluptuous` для валидации сложной конфигурации YAML.
- Внутренняя шина событий (`EventObject`) для координации сенсоров.
- Использование `CoordinatorEntity` для обновления сенсоров, отражающих статус конвейеров загрузки.
