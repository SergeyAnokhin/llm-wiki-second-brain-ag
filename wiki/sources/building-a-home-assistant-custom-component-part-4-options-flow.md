# Building a Home Assistant Custom Component Part 4 Options Flow

**Источник:** Building a Home Assistant Custom Component Part 4 Options Flow.md
**Дата добавления:** 2026-05-05
**Тип:** статья

## Краткое содержание

Четвертая часть руководства разбирает создание [[Options Flow]] — механизма, позволяющего пользователям изменять настройки интеграции уже после ее добавления, не удаляя и не создавая компонент заново.

Рассматривается создание обработчика `OptionsFlowHandler` внутри файла `config_flow.py`. Важным шагом является регистрация `options_update_listener` в методе инициализации (`async_setup_entry`), чтобы при сохранении новых настроек интеграция автоматически перезагружалась (`async_reload`) с обновленными данными. 

## Ключевые утверждения

- В отличие от [[Config Flow]], значения в [[Options Flow]] как правило являются опциональными и могут меняться в любой момент жизни компонента.
- Опции сохраняются в свойстве `options` объекта `config_entry`, которое должно использоваться при инициализации сенсоров.
- Слушатель обновления (update listener) должен быть зарегистрирован, чтобы Home Assistant узнал об изменениях и пересоздал сущности с новыми параметрами.

## Изученные концепции

- [[Options Flow]] — интерфейс для изменения параметров уже добавленной интеграции.
- [[Config Flow]] — первоначальная настройка компонента.
