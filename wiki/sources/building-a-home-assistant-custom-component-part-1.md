# Building a Home Assistant Custom Component Part 1

**Источник:** Building a Home Assistant Custom Component Part 1 Project Structure and Basics.md
**Дата добавления:** 2026-05-05
**Тип:** статья

## Краткое содержание

Эта статья является первой частью руководства по созданию [[Home Assistant Custom Component]]. В ней рассматривается начальная настройка, структура проекта и создание базовой асинхронной интеграции, которая получает данные с использованием внешнего API.

Автор рекомендует использовать шаблон [[Cookiecutter]] для генерации базовой структуры, которая обычно включает папку `custom_components/`, файл `manifest.json` и файл `__init__.py`. В руководстве выделены четыре ключевых шага для реализации компонента: определение зависимостей, настройка схемы конфигурации платформы, асинхронная регистрация сенсоров и реализация логики `async_update` внутри класса сущности.

## Ключевые утверждения

- Использование шаблона [[Cookiecutter]] упрощает создание необходимой структуры директорий и базовых файлов.
- Асинхронное взаимодействие (например, использование `aiohttp` и `gidgethub`) настоятельно рекомендуется вместо синхронных вызовов, чтобы не блокировать цикл событий (event loop) Home Assistant.
- Пользовательский компонент требует расширения класса `homeassistant.helpers.entity.Entity` и реализации метода `async_update` для получения данных.
- Файл `manifest.json` имеет решающее значение для определения метаданных компонента и внешних зависимостей Python.

## Упомянутые сущности

- [[Home Assistant]] — основная платформа для автоматизации умного дома.
- [[Cookiecutter]] — утилита командной строки для генерации файловой структуры компонента из шаблона.
- [[GitHub API]] — внешний сервис, используемый в качестве примера источника данных для пользовательского компонента.
