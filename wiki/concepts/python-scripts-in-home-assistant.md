# Python Scripts in Home Assistant

Альтернатива созданию полноценных компонентов (`custom_components`) или сложных шаблонов Jinja2. Интеграция `python_script` позволяет запускать скрипты на Python прямо из директории `config/python_scripts/`.

## Особенности песочницы
Скрипты выполняются в ограниченной среде выполнения (sandbox):
- **Нет импортов**: Нельзя использовать `import` (ни стандартные библиотеки, ни сторонние).
- **Ограниченный доступ**: Скриптам передается два глобальных объекта:
  - `hass` — для доступа к состояниям (`hass.states.get()`) и вызова сервисов (`hass.services.call()`).
  - `data` — словарь с параметрами, переданными при вызове скрипта (как аргументы сервиса).

## Когда использовать?
Идеально подходит для логики, которая слишком сложна для визуальных автоматизаций YAML или шаблонов Jinja2, но не требует взаимодействия с внешними API, создания новых сущностей или постоянного фонового выполнения (для этого нужны [[Home Assistant Custom Component]]).
