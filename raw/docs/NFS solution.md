Руководство по источнику
Данный текст представляет собой техническое руководство по оптимизации сетевого взаимодействия между сервером Proxmox и хранилищем Synology с целью обеспечения корректной работы режима гибернации. Автор обосновывает переход с протокола SMB на NFS, выделяя его отсутствие состояния (stateless) как ключевое преимущество, которое минимизирует нагрузку на диски и не препятствует переходу системы в сон. Инструкция содержит пошаговый алгоритм настройки, включая активацию служб на сетевом хранилище, разграничение прав доступа и переподключение хранилища в среде виртуализации. В завершение предлагается автоматизировать процесс пробуждения оборудования через скрипты Wake-on-LAN, создавая надежную и энергоэффективную схему для резервного копирования данных.

Лучшая схема для твоей ситуации
NAS спит
    ↓
Proxmox хочет сделать бэкап
    ↓
Pre-backup script: отправляет WOL magic-пакет → ждёт 90 сек
    ↓
NAS просыпается, монтируется NFS
    ↓
Бэкап выполняется
    ↓
Post-backup script: размонтирует NFS
    ↓
NAS снова засыпает через 10 минут

Конкретные шаги
1. Включить NFS на Synology:
DSM → Панель управления → Файловые службы → NFS → включить
2. Дать права на шару:
DSM → Папки общего доступа → Backup → Права NFS → добавить 192.168.1.99 с правами rw
3. В Proxmox добавить NFS storage вместо SMB:
Datacenter → Storage → Add → NFS → указать IP 192.168.1.99
4. Добавить WOL hook для бэкапа:
Создать /etc/vzdump/pre-backup.sh на Proxmox:
#!/bin/bash
if [ "$PHASE" = "job-start" ]; then
    etherwake -i vmbr0 00:11:32:16:b0:6b
    sleep 90
fi
