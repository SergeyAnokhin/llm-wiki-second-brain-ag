Управление hibernation debug
Руководство по источнику
Этот текст представляет собой техническое руководство по мониторингу режима сна на сетевых хранилищах Synology, описывающее методы диагностики работы жестких дисков. Автор объясняет, как активировать отладку и интерпретировать состояние системы, чтобы выявить процессы, препятствующие переходу устройства в состояние покоя. Особое внимание уделяется тому, что внесенные конфигурационные изменения сохраняются после перезагрузки, обеспечивая автоматический запуск логирования. Инструкция структурирована так, чтобы пользователь мог легко отслеживать активность системы через специальные файлы логов и анализировать периоды бездействия оборудования.











```sh
Управление hibernation debug на Synology DS212j
--------------------------------------------------------------------------------
Проверить статус (одна команда)
echo "=== Настройка ===" && \
grep enable_hibernation_debug /etc/synoinfo.conf && \
echo "=== Процесс ===" && \
ps aux | grep syno_hibernation_debug | grep -v grep || echo "процесс НЕ запущен"
Что означает вывод:
enable_hibernation_debug="yes" + строка с процессом → работает
enable_hibernation_debug="no" + пустой вывод → выключен
enable_hibernation_debug="yes" + пустой вывод → настройка включена, но процесс упал (нужно запустить вручную)
--------------------------------------------------------------------------------
Включить debug
# Шаг 1: записать в конфиг
sudo synosetkeyvalue /etc/synoinfo.conf enable_hibernation_debug yes

# Шаг 2: запустить процесс (если не запустился сам)
sudo /bin/sh /usr/syno/sbin/syno_hibernation_debug &

# Шаг 3: проверить
ps aux | grep syno_hibernation_debug | grep -v grep
Логи сразу начнут писаться в:
/var/log/hibernation.log — краткий (кто пишет и idle-периоды)
/var/log/hibernationFull.log — подробный (все операции)
--------------------------------------------------------------------------------
Выключить debug
# Шаг 1: записать в конфиг
sudo synosetkeyvalue /etc/synoinfo.conf enable_hibernation_debug no

# Шаг 2: убить процесс
sudo kill $(ps aux | grep syno_hibernation_debug | grep -v grep | awk '{print $2}')

# Шаг 3: проверить что процесс исчез
ps aux | grep syno_hibernation_debug | grep -v grep || echo "процесс остановлен"
--------------------------------------------------------------------------------
Читать лог в реальном времени
# Только idle-периоды (самое важное)
tail -f /var/log/hibernation.log | grep 'Idle'

# Полный лог с виновниками записей
tail -f /var/log/hibernation.log
--------------------------------------------------------------------------------
Найти максимальный idle за всё время
grep '======Idle' /var/log/hibernation.log | \
  sed 's/[^0-9]//g' | sort -n | tail -5
```
--------------------------------------------------------------------------------
Важное замечание
Изменение через synosetkeyvalue переживает перезагрузку — при следующем старте DSM прочитает конфиг и либо запустит, либо не запустит процесс автоматически. Вручную запускать после ребута не нужно.
